Why WireGuard
Speed
Kernel-level implementation. ~2ms latency between nodes. No userspace overhead. Faster than OpenVPN or IPsec by an order of magnitude.
Security
ChaCha20-Poly1305 encryption. Curve25519 key exchange. Minimal attack surface — ~4,000 lines of code vs 100,000+ for OpenVPN.
Simplicity
Single config file per peer. No certificate authority. No complex PKI. Add a peer in 30 seconds. Remove one in 10.
Failover

If Anastasia goes down, RoadNet WiFi mesh provides backup connectivity between local Pi nodes. 5 access points across the fleet create a local mesh network. Tailscale overlay serves as the secondary fallback — every node has Tailscale installed and connected.

Three layers of connectivity. WireGuard is primary. RoadNet is local backup. Tailscale is the last resort. The fleet stays connected.

Routes

10.8.0.x routes to Alexa's Mac (.28). Fleet nodes can reach each other even across different physical networks. A node in NYC (Anastasia) can talk to a node in the living room (Alice) as if they were on the same switch.

Local → Local
Pi nodes on 192.168.4.x talk directly via LAN. WireGuard adds encryption. RoadNet adds WiFi mesh redundancy.
Local → Remote
Pi nodes reach Anastasia and Gematria (DigitalOcean) through WireGuard tunnels. NAT traversal handled automatically.
Remote → Local
Droplets reach Pi nodes through the hub. Anastasia routes traffic to the correct peer. No exposed ports on home network.
