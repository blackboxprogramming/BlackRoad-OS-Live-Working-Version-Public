Patterns
========

Recommended patterns
--------------------
1. Audit-trail pattern
   Keep an upstream copy when handing off downstream.

2. Live-item pattern
   Move rather than copy when only one active version should exist.

3. Terminal-reference pattern
   When an item reaches stable knowledge, place it in 27-Archive-and-Reference.

4. Example-first pattern
   Keep one demo item trace available so new users can learn by inspection.

5. Layer-discipline pattern
   Change taxonomy only when meaning changes.
   Change workflow step when state changes.
