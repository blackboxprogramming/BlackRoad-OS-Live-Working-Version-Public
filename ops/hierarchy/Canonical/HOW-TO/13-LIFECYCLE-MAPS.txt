Lifecycle maps
==============

This file shows complete end-to-end journeys through the Canonical hierarchy.
Use these maps when you want to understand the whole path of an item, not just one step.

Map 1: Repository to pull request to issue to comment to reference
------------------------------------------------------------------
Use this when engineering work becomes review, then tracked discussion, then durable knowledge.

Flow:
05-Repository
  -> Workflow/23-Ready-to-Move
  -> Workflow/24-Conveyor-Belt
  -> Workflow/25-Transfer
13-Pull-Request
  -> Workflow/01-Intake
  -> Workflow/18-Review
  -> Workflow/23-Ready-to-Move
  -> Workflow/24-Conveyor-Belt
  -> Workflow/25-Transfer
14-Issue
  -> Workflow/01-Intake
  -> Workflow/17-Verify
  -> Workflow/18-Review
  -> Workflow/23-Ready-to-Move
  -> Workflow/24-Conveyor-Belt
  -> Workflow/25-Transfer
15-Comment
  -> Workflow/01-Intake
  -> Workflow/18-Review
  -> Workflow/26-Index
  -> Workflow/27-Archive-and-Reference
16-Ongoing-Reference
  -> Workflow/01-Intake
  -> Workflow/26-Index
  -> Workflow/27-Archive-and-Reference

Meaning:
A code or repository concern matures into review, tracked issue context, discussion, and durable reference.

Map 2: Fast same-layer progression
----------------------------------
Use this when an item stays in one taxonomy layer but advances through work.

Flow:
Layer X
  -> Workflow/01-Intake

  -> Workflow/03-Identify
  -> Workflow/04-Classify
  -> Workflow/05-Scope

Meaning:
The item changes state but not meaning.
No taxonomy handoff occurs yet.

Map 3: Audit-trail handoff pattern
----------------------------------
Use this when you want history preserved at every major crossing.

Flow pattern:
Upstream layer
  -> Workflow/24-Conveyor-Belt
  -> Workflow/25-Transfer
  -> keep upstream copy
Downstream layer
  -> Workflow/01-Intake
  -> continue live work
Trace companion
  -> EXAMPLE-TRACE.txt style history file

Meaning:
The upstream state remains visible while the downstream layer becomes active.

Map 4: Terminal reference lifecycle
-----------------------------------
Use this when an item finishes as durable knowledge rather than active work.

Flow:
Any layer
  -> Workflow/21-Package
  -> Workflow/22-Stage
  -> Workflow/23-Ready-to-Move
  -> Workflow/24-Conveyor-Belt
  -> Workflow/25-Transfer
Reference-oriented layer
  -> Workflow/01-Intake
  -> Workflow/26-Index
  -> Workflow/27-Archive-and-Reference

Meaning:
The item is intentionally preserved for future understanding.

Map 5: Human plus agent collaboration lifecycle
-----------------------------------------------
Use this when a human creates or shapes the item and an agent helps move, document, or verify it.

Flow:
Human creator
  -> creates item from TEMPLATE-ITEM.txt
  -> fills intent and context
Agent or operator
  -> uses move-item.sh to maintain step accuracy
  -> updates docs or trace files when patterns repeat
Auditor or teacher
  -> reviews lineage and explains the flow to others

Meaning:
The hierarchy supports shared work across different roles without changing the structure.

Rule across all maps
--------------------
A lifecycle map may cross layers, but it should never blur the distinction between taxonomy and workflow.
Meaning changes by layer.
State changes by workflow step.
