Operating rhythms
=================

This file defines recurring rhythms for keeping the Canonical hierarchy healthy.
Use these rhythms to prevent drift, clutter, and invisible work.

Daily rhythm
------------
Use this when the hierarchy is active every day.

Daily actions:
- Review new items in `Workflow/01-Intake/`
- Advance stalled items that are ready for the next step
- Check `24-Conveyor-Belt` for handoffs waiting to happen
- Check `25-Transfer` for crossings that need downstream intake
- Correct metadata drift if file contents and locations disagree
- Update or create trace files when major handoffs happened

Daily goal:
Keep flow visible and prevent active work from becoming ambiguous.

Session-start rhythm
--------------------
Use this at the start of a focused work session.

Session-start actions:
- Read `00-HOW-TO-INDEX.txt` if context is needed
- Check the current layer and workflow state before editing anything
- Decide whether the session is creation, movement, audit, or documentation
- Open `11-CHECKLISTS.txt` for the type of action you plan to take
- Use `14-DECISION-TREES.txt` if the next step is unclear

Session-start goal:
Enter the system intentionally instead of making ad hoc moves.

Session-close rhythm
--------------------
Use this before ending a work session.

Session-close actions:
- Make sure item metadata matches folder locations
- Place incomplete work in the most honest workflow step
- Move handoff-ready items through `23`, `24`, or `25` as appropriate
- Add trace notes if the session created meaningful crossings
- Update docs if the session revealed a recurring pattern or confusing term

Session-close goal:
Leave the hierarchy truthful for the next person or agent.

Weekly rhythm
-------------
Use this to maintain quality and structure over time.

Weekly actions:
- Review for duplicate clutter caused by unnecessary copying
- Review `27-Archive-and-Reference` for quality and usefulness
- Look for items stuck too long in one state
- Look for skipped handoffs or missing downstream intake
- Check whether the HOW-TO hub needs a new recipe, glossary term, or decision tree
- Verify shell-safe naming discipline remains intact

Weekly goal:
Keep the model clean, understandable, and teachable.

Monthly rhythm
--------------
Use this for higher-level structural reflection.

Monthly actions:
- Review whether the current docs still match real usage
- Review whether any anti-patterns are recurring
- Review whether lifecycle maps still reflect actual practice
- Decide whether problems should be solved by better usage or a structural change
- Update principles, glossary, or role modes if the system meaning has matured

Monthly goal:
Preserve structural coherence while allowing the system to evolve deliberately.

Handoff rhythm
--------------
Use this whenever work crosses layers.

Handoff actions:
- Confirm the item reached `23-Ready-to-Move`
- Make the queue visible in `24-Conveyor-Belt`
- Cross through `25-Transfer`
- Start the receiving item in the next layer's `01-Intake`
- Decide whether to preserve an upstream copy
- Add or update a trace file if the history matters

Handoff goal:
Make every crossing explicit, auditable, and understandable.

Documentation rhythm
--------------------
Use this when the system teaches something new.

Documentation actions:
- Add glossary entries for recurring new terms
- Add recipes for recurring actions
- Add checklists for recurring mistakes or safeguards
- Add decision trees for recurring moments of uncertainty
- Update the HOW-TO index whenever a new guide is added

Documentation goal:
Turn repeated confusion into stable shared knowledge.

Rule across all rhythms
-----------------------
A rhythm should improve clarity, not add bureaucracy.
If a repeated action does not make the system more truthful, understandable, or durable, simplify it.
