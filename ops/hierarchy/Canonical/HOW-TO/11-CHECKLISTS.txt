Checklists
==========

This file contains short operator checklists for common actions in the Canonical hierarchy.
Use them before moving, handing off, or archiving an item.

Checklist 1: Before creating a new item
---------------------------------------
- Confirm the taxonomy layer is correct.
- Confirm a new item is actually needed.
- Copy `TEMPLATE-ITEM.txt` instead of starting from scratch.
- Use a shell-safe filename like `YYYYMMDD-short-name.item.txt`.
- Fill in `Item ID`, `Title`, `Layer`, `Source`, `Destination Layer`, and `Intent`.
- Place the new file in `Workflow/01-Intake/`.

Checklist 2: Before moving an item inside one layer
---------------------------------------------------
- Confirm the layer meaning has not changed.
- Confirm the target workflow step already exists.
- Decide whether this should be a move, not a copy.
- Keep `Current Step` aligned with the folder.
- Keep `Current Path` aligned with the folder.
- Update `Last Updated`.
- Prefer `move-item.sh` over manual edits.

Checklist 3: Before a cross-layer handoff
-----------------------------------------
- Confirm the item is truly leaving the current layer.
- Confirm `Destination Layer` is correct.
- Move through `23-Ready-to-Move` first.
- Use `24-Conveyor-Belt` as the visible staging point.
- Use `25-Transfer` as the crossing point.
- Decide whether to preserve an upstream copy for audit trail.
- Place the downstream item in the next layer's `Workflow/01-Intake/`.

Checklist 4: Before using copy mode
-----------------------------------
- Confirm there is a real reason to preserve the upstream file.
- Confirm the copy supports traceability, not clutter.
- Confirm the downstream copy is the active one.
- Update `Source` so the lineage is clear.
- Consider adding or updating a trace file.

Checklist 5: Before archiving to reference
------------------------------------------
- Confirm the item is no longer active work.
- Confirm the content is worth preserving.
- Confirm the metadata is truthful.
- Move through `26-Index` before `27-Archive-and-Reference` when possible.
- Ensure the file is understandable to a future reader.
- Avoid using archive as a trash bin.

Checklist 6: Before adding docs to the HOW-TO hub
-------------------------------------------------
- Confirm the term or procedure is recurring enough to deserve documentation.
- Reuse existing vocabulary from `08-GLOSSARY.txt`.
- Prefer examples over vague theory.
- Keep the new doc aligned with existing naming patterns.
- Link the new file from `00-HOW-TO-INDEX.txt`.

Checklist 7: Before changing the structure itself
-------------------------------------------------
- Confirm the problem cannot be solved by better usage alone.
- Confirm you are not mixing taxonomy with workflow.
- Confirm the change preserves shell-safe naming.
- Confirm the change keeps the system teachable.
- Confirm the change scales through repetition rather than special cases.
- Update the HOW-TO docs if the meaning changes.
