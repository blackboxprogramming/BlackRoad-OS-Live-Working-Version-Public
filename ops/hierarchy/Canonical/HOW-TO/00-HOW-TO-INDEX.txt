HOW-TO index
============

This directory is the operating manual for the Canonical hierarchy.
Read these files in order if you are new.

01-QUICKSTART.txt
  Fastest way to create and move an item.

02-WORKING-WITH-LAYERS.txt
  Explains taxonomy layers versus workflow state.

03-MOVING-ITEMS.txt
  Shows manual movement and scripted movement with move-item.sh.

04-HANDOFFS-AND-CONVEYOR.txt
  Explains the meaning of step 24 and step 25.

05-NAMING-RULES.txt
  File naming, layer naming, and path hygiene.

06-PATTERNS.txt
  Recommended patterns for audit trails, duplication, and reference storage.

07-RECIPES.txt
  Exact playbooks for common operations and handoffs.

08-GLOSSARY.txt
  Canonical definitions for the system vocabulary.

09-DESIGN-PRINCIPLES.txt
  The structural rules and reasoning behind the system.

10-ANTI-PATTERNS.txt
  Common mistakes and what to do instead.

11-CHECKLISTS.txt
  Short operator checklists for common actions.

12-ROLE-MODES.txt
  How creators, operators, agents, auditors, and architects should use the system.

13-LIFECYCLE-MAPS.txt
  End-to-end journey maps across layers and states.

14-DECISION-TREES.txt
  Fast branch logic for common operational choices.

15-OPERATING-RHYTHMS.txt
  Daily, weekly, and long-horizon maintenance rhythms.

16-QUALITY-STANDARDS.txt
  What high-quality items, handoffs, traces, and docs look like.

17-TROUBLESHOOTING.txt
  Fast fixes for common operational and structural problems.

18-EXAMPLES-LIBRARY.txt
  Copyable example item shapes and path patterns.

19-BOOTSTRAP-SEQUENCE.txt
  Recommended entry order for first-time or returning users.

20-MANIFESTO.txt
  A short statement of purpose and direction for the system.

21-INDEX-OF-INDEXES.txt
  A meta-map for navigating the whole manual by question type.
