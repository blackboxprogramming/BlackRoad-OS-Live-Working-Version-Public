Bootstrap sequence
==================

This file defines the recommended order for entering and learning the Canonical hierarchy.
Use it when someone is new to the system or returning after a long gap.

Bootstrap goal
--------------
Move from zero context to confident use with the least confusion.

Phase 1: Orient
---------------
Read these first:
1. `00-HOW-TO-INDEX.txt`
2. `08-GLOSSARY.txt`
3. `02-WORKING-WITH-LAYERS.txt`

Why:
These establish the vocabulary and the core distinction between taxonomy and workflow.

Phase 2: See the shape
----------------------
Read these next:
1. `01-QUICKSTART.txt`
2. `13-LIFECYCLE-MAPS.txt`
3. `18-EXAMPLES-LIBRARY.txt`

Why:
These show what the system looks like in action before you get lost in theory.

Phase 3: Learn safe action
--------------------------
Read these next:
1. `03-MOVING-ITEMS.txt`
2. `04-HANDOFFS-AND-CONVEYOR.txt`
3. `11-CHECKLISTS.txt`
4. `14-DECISION-TREES.txt`

Why:
These teach how to move, hand off, and decide correctly.

Phase 4: Learn the logic behind it
----------------------------------
Read these next:
1. `09-DESIGN-PRINCIPLES.txt`
2. `10-ANTI-PATTERNS.txt`
3. `16-QUALITY-STANDARDS.txt`

Why:
These explain why the model works and what failure looks like.

Phase 5: Learn operational depth
--------------------------------
Read these next:
1. `12-ROLE-MODES.txt`
2. `15-OPERATING-RHYTHMS.txt`
3. `17-TROUBLESHOOTING.txt`

Why:
These help you use the system well over time, not just once.

Phase 6: Start with a real artifact
-----------------------------------
Do this next:
- pick a real layer
- copy `TEMPLATE-ITEM.txt`
- create one honest item in `Workflow/01-Intake/`
- move it at least two steps with `move-item.sh`
- read `EXAMPLE-TRACE.txt` for comparison

Why:
The system becomes real only when you operate it.

Fast bootstrap path
-------------------
If you need the shortest useful path, read this order:
1. `08-GLOSSARY.txt`
2. `01-QUICKSTART.txt`
3. `11-CHECKLISTS.txt`
4. `14-DECISION-TREES.txt`
5. `18-EXAMPLES-LIBRARY.txt`

This is the compressed entry path.

Teaching bootstrap path
-----------------------
If you are onboarding someone else, use this order:
1. `08-GLOSSARY.txt`
2. `13-LIFECYCLE-MAPS.txt`
3. `18-EXAMPLES-LIBRARY.txt`
4. `01-QUICKSTART.txt`
5. `EXAMPLE-TRACE.txt`

Why:
Teach by showing the whole picture, then moving into practice.

Rule across the bootstrap sequence
----------------------------------
Do not start by editing the structure.
Start by learning vocabulary, then movement, then examples, then real use.
