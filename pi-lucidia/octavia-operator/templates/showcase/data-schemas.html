🌐 Graph: Domain Routing (Template 003)
All 16 primary domains and their subdomains modeled as a graph. Domain nodes contain metadata (purpose, status, Cloudflare zone), subdomain nodes contain routing details (category, backend, full URL), and routing edges define DNS/worker/service connections.
Example Query: Get all subdomains for blackroad.io
SELECT subdomain, full_url, emoji, category, status FROM template_003_subdomain_nodes WHERE domain_id = 'blackroad-io' ORDER BY category, subdomain; -- Results: -- api | https://api.blackroad.io | 🔌 | api | live -- agents | https://agents.blackroad.io | 🤖 | agents | live -- auth | https://auth.blackroad.io | 🔐 | auth | live -- docs | https://docs.blackroad.io | 📚 | docs | live -- ...
🧠 Emoji Reference Database
All 127 canonical emojis stored as queryable rows with fields for emoji character, name, category (1 of 10), meaning, and usage context. Categories: Layout & Pointers, Context/Platform, Actions, Status/Severity, Resources & Infra, Docs & Learning, People & Roles, Time & Scheduling, Observability, Meta/Vibe.
Sample Emoji Records
SELECT emoji, name, category, meaning FROM emoji_reference; -- 🚀 | Deploy/launch | Actions | Ship to environment -- ✅ | OK/done | Status/Severity | Succeeded, safe, verified -- ⛅️ | Cloudflare | Context/Platform | Workers, R2, D1, Wrangler -- 🧱 | Database | Resources/Infra | D1 tables, infra blocks -- 💡 | Tip/insight | Docs & Learning | Pro tips, best practice -- 🔥 | Epic done | Meta / Vibe | Big milestone achieved
📊 Queryable Views
Four pre-built views for common queries across the template system:
-- Templates grouped by category with counts CREATE VIEW templates_by_category AS SELECT category, COUNT(*) AS template_count FROM template_catalog GROUP BY category; -- Top 10 most-used templates CREATE VIEW most_used_templates AS SELECT name, emoji, usage_count FROM template_catalog WHERE usage_count > 0 ORDER BY usage_count DESC LIMIT 10; -- Template dependency graph CREATE VIEW template_dependency_graph AS SELECT t.template_id, t.name AS template_name, d.depends_on_id AS depends_on_template_id, dp.name AS depends_on_template_name FROM template_dependencies d JOIN template_catalog t ON t.template_id = d.template_id JOIN template_catalog dp ON dp.template_id = d.depends_on_id; -- Domain statistics with live/pending counts CREATE VIEW domain_stats AS SELECT domain, emoji, purpose, COUNT(*) AS subdomain_count, SUM(CASE WHEN status = 'live' THEN 1 ELSE 0 END) AS live, SUM(CASE WHEN status = 'pending' THEN 1 ELSE 0 END) AS pending FROM template_003_domain_nodes GROUP BY domain, emoji, purpose;
