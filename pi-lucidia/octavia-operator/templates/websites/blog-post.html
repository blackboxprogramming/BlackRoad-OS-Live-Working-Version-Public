Why agents should never see your API keys

Alexa Amundson / Feb 28, 2026 / 8 min read

The most common pattern in AI-powered applications is also the most dangerous: embedding API keys directly in agent code. It works. It ships. And it creates a blast radius that grows with every agent you deploy.

At BlackRoad OS, we took a different approach. Our tokenless gateway architecture ensures that no agent — not one of our 30,000 — ever sees a provider API key. Here's why, and how.

The problem with embedded keys

When an agent carries its own API key, three things are true:

  • Every agent is a potential leak vector
  • Key rotation requires touching every agent
  • There's no central audit trail for provider calls

At small scale, this is manageable. At 30,000 agents distributed across Raspberry Pis and cloud droplets, it's a liability.

The gateway pattern

Instead of giving agents keys, we give them a gateway address:

[Agent CLI] ---> [BlackRoad Gateway :8787] ---> [Ollama/Claude/OpenAI]

The gateway is the only component that holds provider credentials. Agents authenticate to the gateway via the trust boundary — not via API keys.

The gateway is not a proxy. It's a trust boundary. That distinction matters.

Verification

We run verify-tokenless-agents.sh on every commit. It scans all agent code for forbidden strings — API key patterns, provider URLs called directly, hardcoded tokens. If it finds anything, the commit is rejected.

$ ./blackroad-core/scripts/verify-tokenless-agents.sh
Scanning 30,000 agent configs...
0 violations found.
All agents are tokenless.

Why this matters beyond security

The tokenless pattern gives us three capabilities for free:

  • Central logging — every provider call flows through one point, so we get a complete audit trail
  • Instant rotation — change a key once at the gateway, and every agent picks it up immediately
  • Provider switching — swap from OpenAI to local Ollama without touching a single agent

Architecture decisions compound. The tokenless gateway was a day of work that saved us months of security debt.
