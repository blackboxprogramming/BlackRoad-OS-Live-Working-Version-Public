Introduction

BlackRoad OS is an AI-native developer platform for running, orchestrating, and managing AI agents on your own hardware. This guide covers everything from installation to deploying a fleet of 30,000 agents.

Note

BlackRoad OS runs on macOS, Linux, and Raspberry Pi OS. Windows support is available through WSL.

What is BlackRoad OS?

BlackRoad OS is three things:

  • A CLI — 84 tools unified under the br command for deployment, monitoring, agent management, and more
  • An agent runtime — infrastructure for running up to 30,000 AI agents across distributed hardware
  • A trust architecture — the tokenless gateway ensures agents never handle API keys directly

Key Concepts

Concept Description
Agent An autonomous unit with identity, email, memory, and capabilities
Gateway The tokenless trust boundary between agents and AI providers
PS-SHA Hash-chained, append-only memory journals for tamper-evident logging
CECE Portable AI identity system that persists across sessions and providers
Fleet The collection of all agents across all hardware nodes

Architecture overview

[Agent CLIs] ---> [Gateway :8787] ---> [Ollama / Claude / OpenAI]
                        |
                  [PS-SHA Memory]
                        |
                 [CECE Identity DB]

The gateway is the single point of provider communication. Agents call the gateway; the gateway holds the secrets. Every call is logged. Every action is hash-chained.

Tip

Run br geb oracle to ask the system what it cannot know about itself. Yes, we built Hofstadter into the CLI.
