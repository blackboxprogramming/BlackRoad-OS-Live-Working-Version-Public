03 — The Missing Sandbox

There Is No CodePen for Physics

Web developers have CodePen, JSFiddle, and Replit. Data scientists have Jupyter notebooks. Designers have Figma. But physicists, engineers, chemists, and biologists? They have million-dollar simulation suites locked behind enterprise contracts, or they have nothing.

// BlackRoad Quantum Sandbox — bridge simulation
import { Scene, Material, Force } from '@blackroad/quantum';

const bridge = new Scene('truss-bridge');
bridge.addMaterial('steel-A36', {
  yield: 250e6, // Pa
  elastic: 200e9, // Pa
  density: 7850, // kg/m^3
});

bridge.addLoad('center', new Force({
  magnitude: 500e3, // 500 kN
  direction: [0, -1, 0],
  type: 'distributed',
}));

const results = await bridge.simulate({
  solver: 'fem-adaptive',
  meshSize: 0.05,
  nonlinear: true,
});

console.log(results.maxStress); // 187.3 MPa — safe!
console.log(results.maxDeflection); // 12.4 mm
bridge.visualize('stress-heatmap');

05 — Architecture

WebAssembly Simulations in the Browser

No plugins. No downloads. No GPU requirements. BlackRoad Quantum compiles physics engines to WebAssembly and runs them at near-native speed in any modern browser. The architecture is designed for real-time collaboration and infinite scalability.

Interface
React-like component system with drag-and-drop scene builder. Real-time parameter sliders. 3D viewport with WebGL2 rendering. Code editor with IntelliSense for simulation APIs.
Simulation
WebAssembly-compiled physics engines: Bullet (rigid body), OpenFOAM (CFD), LAMMPS (molecular dynamics). Adaptive time-stepping, parallel solver decomposition, GPU compute via WebGPU when available.
Data
IndexedDB for local state persistence. Streaming data format for real-time visualization. HDF5-compatible export. Git-like versioning of simulation states with branching and merging.
Compute
Client-side for small simulations (instant feedback). Web Workers for parallel decomposition. Optional cloud burst for large-scale runs — your data never leaves your machine unless you choose.
Collaboration
WebRTC peer-to-peer for real-time co-simulation. CRDT-based conflict resolution. Shareable URLs with embedded state. Fork any public simulation with one click. Comment and annotate in 3D space.
