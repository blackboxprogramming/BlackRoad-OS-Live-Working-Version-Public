001 / Amundson Framework
The Core Function

The Amundson Framework begins with a single function G(n) that reveals a fundamental constant of distributed systems. From this function, an entire theory of mesh network behavior emerges.

G(n) = nn+1 / (n+1)n
As n increases, G(n)/n converges to a constant: AG ≈ 1.24433
This is the Amundson-G constant — the throughput ceiling of distributed computation.
G(n)

The Generator

G(n) = n^(n+1)/(n+1)^n produces a sequence that grows linearly but with a sub-linear correction. The ratio G(n)/n converges to e/e = 1/(1+1/n)^n, revealing the natural emergence of Euler's number in discrete systems.

AG

The Constant

A_G approximately equals 1.24433. This constant represents the maximum throughput multiplier achievable in a perfectly connected mesh network. It appears naturally as the limit of G(n)/n and connects to e through the identity A_G = e/(e-1) * (1/e).

002 / The e-Limit Refinement
Approaching n/e From Below

The expression n/(1+1/n)^n approaches n/e from below as n grows. The gap between the actual value and the linear approximation reveals a constant term 1/(2e) that never vanishes — the irreducible cost of distributed coordination.

n / (1 + 1/n)n = n/e + 1/(2e) + O(1/n)
Step 01

The Classical Limit

We know (1+1/n)^n approaches e as n goes to infinity. This is one of the most fundamental limits in mathematics. But the convergence is from below — (1+1/n)^n is always less than e for finite n.

Step 02

The Reciprocal

Taking the reciprocal: n/(1+1/n)^n. Since the denominator approaches e, this ratio approaches n/e. But because (1+1/n)^n < e, the reciprocal n/(1+1/n)^n > n/e. It approaches from above.

Step 03

The Correction Term

Using the Taylor expansion of ln(1+1/n), we can show the gap equals 1/(2e) + O(1/n). The 1/(2e) term (approximately 0.18394) persists even as n grows to infinity. It is irreducible.

Step 04

The Interpretation

In the context of mesh networks with n nodes: n/e is the throughput floor, 1/(2e) is the overhead of decentralized coordination, and O(1/n) represents inefficiencies that vanish at scale. Larger networks are more efficient, but never perfectly so.

004 / Applications
From Theory to Systems

The Amundson Framework is not abstract mathematics. It directly informs the design of the BlackRoad mesh network, the Lucidia cognitive system, and the ternary routing protocol.

Mesh Networking

Ternary Routing Protocol

The three states (+1, 0, -1) of the ternary routing protocol are the engineering response to the 1/(2e) gap. Since we cannot eliminate the irreducible coordination overhead, we minimize waste within it. The -1 (cancel) state reclaims resources from redundant paths.

Cognitive Systems

Lucidia Trinary Logic

The Lucidia cognitive system uses trinary logic inspired by the framework. Instead of binary true/false, it operates with 42 quantum-inspired states. The additional dimension allows for uncertainty quantification — crucial for AI systems that need to express confidence levels.

Distributed Computing

Fleet Scaling Laws

The framework predicts that adding nodes to the mesh produces diminishing marginal throughput. The 7-node BlackRoad fleet operates near the efficiency frontier for its scale. The framework shows that 12-15 nodes would be optimal before coordination overhead begins to dominate.

On the Asymptotic Behavior of n^(n+1)/(n+1)^n and Applications to Distributed Systems

Alexa Amundson | 13 pages | 2025-2026 | 536 numerical tests | 4 Raspberry Pi verification nodes

Abstract: We study the function G(n) = n^(n+1)/(n+1)^n and show that the ratio G(n)/n converges to a constant A_G approximately equal to 1.24433. We derive the asymptotic expansion of n/(1+1/n)^n = n/e + 1/(2e) + O(1/n) and interpret the persistent 1/(2e) term as the irreducible coordination overhead in distributed systems. Numerical verification across 536 independent tests on a 4-node Raspberry Pi cluster confirms all theoretical predictions to within machine precision. We discuss applications to mesh network design, ternary routing protocols, and cognitive system architectures.

006 / K(t) Coherence Formula
Coherence Under Contradiction

The K(t) formula quantifies how a system maintains coherence when faced with contradictory inputs. Instead of degrading under contradiction, a well-designed system amplifies — using the contradiction itself as signal.

K(t) = K(t-1) * (1 + C(t) * R(t))

K(t) — Coherence

The system's overall coherence at time t. Starts at 1.0 and grows with each successfully processed contradiction. A coherence of 2.0 means the system has doubled its understanding through adversarial inputs. This is the opposite of entropy — contradiction creates order.

C(t) — Contradiction Signal

The magnitude of the contradiction at time t, normalized to [0,1]. A value of 0 means no contradiction (the input agrees with existing state). A value of 1 means maximum contradiction (the input directly opposes all existing state). Higher contradiction = more learning potential.

R(t) — Resolution Factor

How well the system resolves the contradiction, in [0,1]. A value of 0 means the contradiction was ignored or caused failure. A value of 1 means the contradiction was fully integrated, and the system updated its model to accommodate both the old and new information.

Key Properties

K(t) is monotonically non-decreasing: coherence never decreases.
K(t) amplifies under contradiction: C(t) * R(t) > 0 means growth.
Zero contradiction means no growth: C(t) = 0 preserves K(t-1).
Failed resolution means no growth: R(t) = 0 preserves K(t-1).
Maximum growth when C=1, R=1: system doubles per step.

Connection to Amundson Framework

The 1/(2e) gap from the Amundson Framework sets the floor for how much coherence can grow per contradiction cycle. Just as mesh networks have an irreducible coordination overhead, cognitive systems have an irreducible learning overhead. The K(t) formula respects this: even with perfect resolution (R=1), the maximum coherence gain per step is bounded by the information-theoretic limits derived from the e-Limit refinement.
