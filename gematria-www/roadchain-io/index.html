Features

Sovereign by Design

Identity and state management that uses tools developers already know. No new protocols to learn. No tokens to buy.

GN

Git-Native Blockchain

Git is already a Merkle DAG -- the same data structure underlying every blockchain. RoadChain makes this explicit. Your commit history IS your ledger. Your branches ARE your forks. git push IS your broadcast. No new tooling required. Every Git client is a RoadChain client.

ID

Identity Verification

RoadChain-SHA2048 extends standard Git SHA hashing to 2048-bit identity proofs. Your SSH key becomes your sovereign identity. GPG signatures become attestations. No centralized identity provider. No OAuth tokens. Your cryptographic keys are your credentials.

TX

Commit-as-Transaction

Every Git commit is a signed, timestamped, content-addressed transaction. The commit message is the transaction memo. The diff is the state change. The parent hash is the chain link. Revert a commit? That is a refund. Merge a branch? That is a settlement.

SM

Sovereign State Management

Your repo, your state. No consensus mechanism needed when you own the chain. For multi-party chains, RoadChain uses Git's built-in merge conflict resolution as its consensus protocol. Disagreements are resolved the way developers already resolve them: with code review.

How It Works

Get Started in 4 Steps

1

Initialize Your Chain

Create a new repository on RoadCode (our self-hosted Gitea instance) or any Git server you control. The first commit is your genesis block.

git init my-chain && cd my-chain && git commit --allow-empty -m "genesis"
2

Sign Your Identity

Configure GPG signing for all commits. Your GPG key becomes your RoadChain identity. Every commit you sign is cryptographically provable as yours.

git config commit.gpgsign true
3

Commit State Changes

Every commit is a state transition. Store data, configuration, or identity claims as files. The diff is the state change. The hash is the proof.

git commit -m "state: update identity claim"
4

Verify the Chain

Verify any chain by cloning it and checking the commit signatures. The full history is the full proof. No block explorer needed -- git log IS the explorer.

git log --show-signature
