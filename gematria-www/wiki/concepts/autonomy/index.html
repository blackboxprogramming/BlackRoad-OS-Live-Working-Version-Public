Concept · Self-Healing
Autonomy
Self-Healing Infrastructure
The fleet heals itself. Heartbeat every 60 seconds, heal every 5 minutes, watchdog every 30 seconds. Services auto-restart without human intervention.
The Stack
Heartbeat
1-minute cron. Checks if the node is alive. Reports status to stats-proxy. The pulse of the fleet — if this stops, something is wrong.
Heal
5-minute cron. Checks stats-proxy and Ollama services. If a service is down, it auto-restarts. No tickets, no pages, no humans.
Watchdog
30-second timer on Alice. Monitors the Redis task queue via task_watchdog.py. Catches stuck tasks before they cascade.
Where it runs
Cecilia + Octavia — blackroad user cron. Heartbeat (1min) + Heal (5min). Both nodes run identical autonomy scripts.
Alice — blackroad-watchdog.timer (30s). Redis task queue monitoring. The gateway watches everything.
Lucidia — brnode-heartbeat.timer (5min). Lightweight heartbeat via /opt/blackroad/bin/brnode heartbeat.
Logs

All autonomy activity logged to ~/.blackroad-autonomy/cron.log. Power monitor runs on all nodes via /opt/blackroad/power-monitor.sh (cron every 5 minutes), logging to /var/log/blackroad-power.log. Every restart, every health check, every recovery — recorded.

Philosophy

“Systems that need heroes are badly designed.” Auto-remediation means 3am problems fix themselves. The goal is zero human intervention for routine failures. If a service crashes, the fleet notices within 60 seconds and heals within 5 minutes. No alerts, no wake-ups, no firefighting.
