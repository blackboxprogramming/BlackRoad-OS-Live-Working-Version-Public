Concept · Programming Language
RoadC
The Language of the Road
A custom programming language with Python-style indentation. Lexer → Parser → Interpreter. Functions, recursion, pattern matching, 3D space primitives.
What is RoadC?

A custom language designed for BlackRoad OS. Python-style colon + INDENT/DEDENT blocks define scope — no braces, no semicolons. The keyword is fun (not fn), with let/var/const for bindings, match for pattern matching, spawn for concurrency, and space for 3D primitives. Executed by a tree-walking interpreter.

Features
Functions & Recursion
First-class functions with fun keyword. Full recursion support. Closures capture their environment.
If / Elif / Else
Branching with Python-style indentation. No parentheses required. Clean, readable control flow.
While & For Loops
Iteration primitives. While loops for conditions, for loops for sequences. Break and continue supported.
Pattern Matching
match expressions for structural pattern matching. Destructure data, guard clauses, exhaustive checking.
Spawn
Concurrency primitive. spawn launches parallel execution. Designed for multi-agent coordination.
Space (3D Primitives)
space keyword for 3D scene description. Vectors, transforms, meshes — language-level spatial reasoning.
Example
fun fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)
Architecture
1
Lexer
Tokenizes source code into tokens. Handles INDENT/DEDENT, keywords, literals, operators.
2
Parser
Builds an abstract syntax tree (AST) from the token stream. Recursive descent parsing.
3
Interpreter
Tree-walks the AST and executes. Environment-based scoping. Direct evaluation.
Stats

Source at ~/roadc. All working: functions, recursion, if/elif/else, while, for, strings, integers, floats. Full pipeline — lexer, parser, interpreter — built from scratch.
