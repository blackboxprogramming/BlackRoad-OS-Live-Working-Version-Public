Concept · Metrics
KPI System
Measuring What Matters
Daily automated collection of fleet metrics. GitHub commits, Gitea repos, fleet health, LOC counts. The blackroad-os-kpis monorepo.
What it tracks
GitHub
Commits, pull requests, active repos. Tracks contribution velocity across the entire blackboxprogramming organization.
Gitea
207 repos across 7 orgs on Octavia. Mirrors, forks, and original repos. Size and activity tracking.
Fleet
SSH probe — CPU, memory, disk, uptime per node. Knows which Pis are alive and which are struggling.
Autonomy
Heartbeat and heal success rates. How often do services self-recover vs. need intervention?
LOC
Lines of code across all repos. Shell, Python, TypeScript, HTML, CSS — the full stack measured.
How it runs

Daily cron at 6 AM triggers all 5 collectors. GitHub Actions runs a parallel collection job. Slack notification fires at 6:05 AM with a summary. First collection on 2026-03-12: 343 commits, 102 repos, 100K+ lines of code, 3 of 4 fleet nodes online, 27 Ollama models running.

Architecture

Monorepo at ~/blackroad-os-kpis → github.com/blackboxprogramming/blackroad-os-kpis. Each collector is a standalone script. Data stored in SQLite. JSON export for dashboards and Slack. Designed to run on any node in the fleet.
