Concept · Autonomous Recovery
Self-Healing
Every node monitors itself and its neighbors. Heartbeat every minute, heal every five. Auto-restart of critical services. Zero human intervention for common failures.
The Cycle

Two cron loops run on every node. The heartbeat fires every 60 seconds — a pulse that says "I'm alive." The heal cycle fires every 5 minutes — checking stats-proxy, Ollama, and critical services. If something's down, it comes back up. No tickets, no pages, no humans.

*/1 * * * * heartbeat.sh && */5 * * * * heal.sh
cron · heartbeat (1m) + heal (5m) · every node
How it works

Autonomy scripts run as blackroad user cron jobs on Cecilia and Octavia. Alice runs a watchdog timer every 30 seconds via systemd. Lucidia uses brnode heartbeat on a 5-minute timer. Each layer is independent — if one fails, the others still heal.

heartbeat.alive — node confirms it is running
heal.check — critical services inspected
service.restart — downed service auto-restarted
watchdog.trigger — Alice task queue anomaly detected
Why it matters
Uptime
Common failures resolve in under 5 minutes. No waiting for a human to wake up, SSH in, and restart a service. The fleet heals itself while you sleep.
Layered Defense
Cron jobs, systemd watchdogs, and brnode heartbeats — three independent recovery mechanisms. If one layer fails, the next catches it.
Observable
Every heal action is logged. Every restart is recorded. The KPI system tracks fleet health daily. You know exactly what healed and when.
The Philosophy

Infrastructure that needs you to fix it is infrastructure that owns you. Self-healing flips the relationship. The machines serve you, not the other way around.
