Browser Mesh
Every link is a node. Browser tabs as compute nodes via WebGPU + WASM + WebRTC. Pi fleet as permanent backbone, browser nodes as elastic scale. The network grows with every visitor.
The Architecture

The Pi fleet (52 TOPS) forms the permanent backbone — always on, always available. Browser nodes join when users visit blackroad.io/chat or embed mesh.js. WebRTC handles peer-to-peer data channels. WebGPU runs inference in the browser. WASM bridges the gap. Every tab is a node. Every visitor adds capacity.

How it works

Browser nodes connect via WebRTC signaling to the Pi backbone. Inference tasks are sharded across available nodes — the mesh scheduler routes work to the fastest available compute. Results aggregate back through the mesh. The OpenAI-compatible API at 50% price is powered by this distributed compute fabric.

mesh.join — browser tab registers as compute node
webrtc.connect — peer-to-peer data channel established
webgpu.infer — inference shard executed in browser
mesh.aggregate — results collected from distributed nodes
Why it matters
Elastic Scale
The Pi fleet handles baseline load. When demand spikes, browser nodes absorb it. No auto-scaling groups, no Kubernetes, no cloud bills. Scale is a function of traffic.
Distribution
blackroad.io/chat gives free AI to anyone. mesh.js lets any site become a node. Shareable inference links turn URLs into compute endpoints. The mesh is the product.
Revenue
OpenAI-compatible API at 50% price. 70/30 compute provider split. $199 hardware kits for permanent nodes. The mesh pays for itself and then some.
The Philosophy

Every link is a node. The web was built for documents. We're rebuilding it for compute. Every browser tab has a GPU. Every visitor has spare cycles. The mesh just connects them.
