## 4. Data Security Measures

### A. Technical Security Controls

**Encryption:**
- **At Rest:** AES-256 encryption for all databases and file storage
- **In Transit:** TLS 1.3 for all web traffic and API communications
- **Backups:** Encrypted backups with 3-2-1 strategy (3 copies, 2 media, 1 offsite)

**Access Controls:**
- **Multi-Factor Authentication (MFA):** Required for all account access
- **Role-Based Access Control (RBAC):** Employees have access only to data needed for their role
- **Least Privilege Principle:** Minimum necessary permissions granted
- **Session Management:** 30-minute timeout for inactive sessions

**Network Security:**
- **Firewall:** Cloudflare Web Application Firewall (WAF)
- **DDoS Protection:** Cloudflare DDoS mitigation
- **Intrusion Detection:** Real-time monitoring for suspicious activity
- **Vulnerability Scanning:** Quarterly penetration testing

### B. Organizational Security Controls

**Employee Training:**
- Annual cybersecurity and privacy training (required for all employees)
- Phishing simulation exercises (quarterly)
- Incident response drills (annually)

**Background Checks:**
- All employees undergo background checks before hire
- Fingerprinting and FINRA registration (for associated persons)

**Confidentiality Agreements:**
- All employees sign confidentiality and non-disclosure agreements
- Terminated employees lose access immediately upon separation

### C. Physical Security

**Data Centers:**
- SOC 2 Type II certified cloud infrastructure (AWS or equivalent)
- 24/7 physical security and surveillance
- Redundant power and network connectivity

**Office Security:**
- Locked file cabinets for physical documents
- Shred sensitive documents before disposal
- Visitor logs and badge access controls

### D. Incident Response

In the event of a data breach:

1. **Immediate Response (0-24 hours):**
   - Contain the breach (isolate affected systems)
   - Notify senior management and legal counsel
   - Begin forensic investigation

2. **Notification (varies by jurisdiction):**
   - **Customers:** Notify affected individuals without unreasonable delay
   - **SEC/FINRA:** Notify if material impact on operations
   - **State Attorneys General:** Comply with state breach notification laws (typically 30-60 days)

3. **Remediation:**
   - Implement corrective measures to prevent recurrence
   - Offer credit monitoring services to affected individuals (if applicable)
   - Update security controls and policies

## 7. Identity Theft Prevention (Red Flags Rule)

BlackRoad OS, Inc. has implemented an **Identity Theft Prevention Program** in compliance with **Regulation S-ID (16 CFR § 681.2)** - "Red Flags Rule."

### A. Purpose

Detect, prevent, and mitigate identity theft when opening accounts or conducting transactions.

### B. Red Flags We Monitor

**Account Opening Red Flags:**
- Fraudulent documents (altered ID, fake passport)
- Photo on ID does not match applicant
- Information inconsistent with credit bureau (different address, SSN)
- Applicant cannot provide authenticating information

**Account Activity Red Flags:**
- Sudden change of address followed by request for new debit card
- Account accessed from unusual geographic location (foreign country)
- Mail returned as undeliverable
- Customer reports they did not authorize account opening

**Transaction Red Flags:**
- Large, unusual transaction inconsistent with customer profile
- Transaction pattern inconsistent with customer history
- Rapid movement of funds (deposit → immediate withdrawal)

### C. Response Procedures

When a red flag is detected:

1. **Alert:** System flags transaction or account for review
2. **Investigation:** Compliance officer investigates within 24 hours
3. **Verification:** Contact customer via **verified phone number** (not number provided by potential fraudster)
4. **Action:** Freeze account, reject transaction, request additional documentation
5. **Documentation:** Record red flag incident in compliance log

### D. Program Oversight

- **Program Administrator:** Chief Compliance Officer (Alexa Amundson)
- **Annual Review:** Red Flags program reviewed and updated annually
- **Board Reporting:** Report identity theft incidents to board quarterly
