Troubleshooting
===============

This file provides fast fixes for common problems in the Canonical hierarchy.
Use it when something looks wrong, stalled, duplicated, or confusing.

Problem 1: Metadata does not match the folder
---------------------------------------------
Symptoms:
- `Current Step` disagrees with the folder name
- `Current Path` is stale
- `Layer` does not match the taxonomy layer

Fix:
- Decide whether the file or the metadata is the truth
- If the location is right, update the metadata
- If the metadata is right, move the file to the correct folder
- Prefer `move-item.sh` if the item is supposed to change step

Prevention:
Use the script instead of manual moves when possible.

Problem 2: The item is in the wrong workflow step
-------------------------------------------------
Symptoms:
- The item was placed too early or too late in the workflow
- The file is honest about content but not honest about state

Fix:
- Keep the taxonomy layer the same
- Move the item to the correct step
- Update metadata if needed
- Do not copy unless you explicitly want history

Prevention:
Use `11-CHECKLISTS.txt` before moving items.

Problem 3: The item is in the wrong taxonomy layer
--------------------------------------------------
Symptoms:
- The file meaning no longer matches the layer
- The item became a different kind of thing

Fix:
- Confirm the new meaning really requires a different layer
- Move through the visible handoff path when appropriate
- Start the downstream file in `01-Intake`
- Update `Destination Layer`, `Source`, and trace information if useful

Prevention:
Use `14-DECISION-TREES.txt` to test same-layer versus cross-layer moves.

Problem 4: Too many duplicate copies exist
------------------------------------------
Symptoms:
- Multiple nearly identical files exist across steps
- It is unclear which copy is active

Fix:
- Decide which copy is the live one
- Remove or consolidate copies that do not add audit value
- Keep upstream copies only where lineage matters
- Add a trace file if the remaining copies need explanation

Prevention:
Copy only when the history is intentionally useful.

Problem 5: A handoff was skipped
--------------------------------
Symptoms:
- The item appears in a downstream layer with no visible crossing
- There is no `24-Conveyor-Belt` or `25-Transfer` record

Fix:
- If traceability matters, reconstruct the missing handoff with an audit note or trace file
- Resume future crossings with the visible bridge
- Do not normalize the shortcut into a habit

Prevention:
Treat `24` and `25` as first-class states, not optional decoration.

Problem 6: Archive-and-Reference is getting messy
-------------------------------------------------
Symptoms:
- Step 27 contains unfinished work
- Archived items are low quality or contextless
- The reference layer feels like a trash pile

Fix:
- Remove or relocate unfinished items
- Improve context for references worth keeping
- Re-index items through `26-Index` when needed
- Keep only durable, understandable material in step 27

Prevention:
Archive only for durable knowledge, not cleanup.

Problem 7: A new case does not fit cleanly
------------------------------------------
Symptoms:
- A user wants a new folder or state because the current model feels awkward
- The case seems novel or edge-shaped

Fix:
- First test whether better item metadata solves it
- Then test whether an existing workflow state already fits
- Then test whether the issue is documentation, not structure
- Change the structure only if the meaning truly cannot be expressed honestly

Prevention:
Prefer composition and discipline over novelty.

Problem 8: The HOW-TO hub feels incomplete
------------------------------------------
Symptoms:
- The same confusion keeps returning
- People ask the same operational question repeatedly

Fix:
- Add a glossary entry for recurring vocabulary confusion
- Add a recipe for recurring action patterns
- Add a checklist for recurring safeguards
- Add a decision tree for recurring branching questions
- Update the index immediately

Prevention:
Convert repeated confusion into stable documentation.

Problem 9: The hierarchy feels bureaucratic
-------------------------------------------
Symptoms:
- Users feel slowed down by the process
- Too much ceremony is happening without adding clarity

Fix:
- Identify which steps are adding truth and which are adding noise
- Keep the parts that improve clarity, traceability, or teachability
- Simplify usage before changing structure
- Preserve the distinction between meaning and state

Prevention:
The system should be intricate, not wasteful.

Rule across all troubleshooting
-------------------------------
Recover truth first.
When something is wrong, restore the most honest relationship between meaning, state, metadata, and location.
