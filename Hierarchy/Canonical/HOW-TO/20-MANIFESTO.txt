Manifesto
=========

This directory is not just a file tree.
It is an attempt to make structure and motion legible at the same time.

Most directories only store things.
This one is meant to explain what a thing is, where it is, how it moves, and why that movement matters.

We believe meaning and state should not be confused.
That is why taxonomy and workflow are separate.
A repository is not a review state.
A pull request is not a queue lane.
A comment is not an archive.

We believe movement should be visible.
That is why handoff is explicit.
A crossing should not disappear into assumption.
The conveyor belt exists because transitions matter.

We believe complexity should be intricate, not chaotic.
That is why the system repeats the same motion model across layers.
The goal is not novelty.
The goal is legibility at scale.

We believe documentation should live with the system it explains.
That is why the HOW-TO hub is inside the hierarchy.
A good operating model should teach itself.

We believe examples matter more than abstraction alone.
That is why there are templates, traces, maps, recipes, checklists, and example paths.
A system becomes trustworthy when a newcomer can inspect it and understand it.

We believe traceability is a feature, not clutter, when it reveals truth.
An audit trail is valuable when it explains lineage.
Reference is valuable when it preserves meaning.
Archive is valuable when it remains readable.

We believe a directory can be an operating system for thought.
Not software in the narrow sense, but a disciplined environment where work, reference, and handoff stay intelligible.

The point of this hierarchy is not to make work heavier.
The point is to make work more honest.
More visible.
More teachable.
More durable.

If this system succeeds, it will help people and agents coordinate without losing meaning.
It will turn folders into memory, movement, and method.
