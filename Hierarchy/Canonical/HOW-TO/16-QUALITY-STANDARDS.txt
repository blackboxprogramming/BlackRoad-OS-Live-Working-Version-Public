Quality standards
=================

This file defines what good looks like inside the Canonical hierarchy.
Use these standards to judge items, handoffs, traces, docs, and archived references.

Standard 1: A good item
-----------------------
A good item:
- has a clear `Item ID` and `Title`
- lives in the correct taxonomy layer
- lives in the correct workflow step
- has truthful `Current Step` and `Current Path`
- has a useful `Intent`
- names a plausible `Destination Layer` when relevant
- has metadata that matches the file's actual situation

A weak item:
- has vague naming
- has stale metadata
- sits in a folder that does not match its contents
- does not explain what it is trying to do

Standard 2: A good move
-----------------------
A good move:
- changes the location intentionally
- keeps the layer stable when meaning did not change
- updates metadata to match the new folder
- uses `move-item.sh` when possible
- leaves only one active copy unless duplication is intentional

A weak move:
- leaves mismatched metadata
- changes both meaning and state without clarity
- creates accidental duplicate versions

Standard 3: A good handoff
--------------------------
A good handoff:
- makes readiness visible through `23-Ready-to-Move`
- makes queueing visible through `24-Conveyor-Belt`
- makes crossing visible through `25-Transfer`
- places the downstream item in `01-Intake`
- preserves an upstream copy only when useful
- keeps lineage understandable

A weak handoff:
- skips visibility
- jumps layers with no trace
- loses the relationship between upstream and downstream work

Standard 4: A good trace file
-----------------------------
A good trace file:
- lists the important path sequence clearly
- uses one file per item or journey where helpful
- is easy to read at a glance
- helps an auditor or teacher reconstruct the lifecycle quickly

A weak trace file:
- is incomplete
- is inconsistent with the actual file locations
- exists without adding clarity

Standard 5: A good archived reference
-------------------------------------
A good archived reference:
- is no longer active work
- is worth preserving for future understanding
- is readable without requiring missing context
- has honest metadata
- sits in `27-Archive-and-Reference` because it is durable, not because it is inconvenient

A weak archived reference:
- is unfinished and abandoned
- is low quality or contextless
- is being used as a trash pile

Standard 6: A good HOW-TO document
----------------------------------
A good HOW-TO document:
- solves a recurring need
- uses the canonical vocabulary
- is concrete, not vague
- links naturally from the index
- complements the rest of the manual instead of duplicating it badly

A weak HOW-TO document:
- repeats existing material without adding clarity
- invents new vocabulary unnecessarily
- is too abstract to help real use

Standard 7: A good structure change
-----------------------------------
A good structure change:
- preserves the difference between taxonomy and workflow
- keeps the system shell-safe and teachable
- scales by repeating patterns instead of adding one-off cases
- updates docs when meaning changes

A weak structure change:
- adds novelty without solving a real representational problem
- makes the tree less predictable
- forces users to rely on memory instead of the manual

Rule across all standards
-------------------------
Quality means truthfulness, clarity, and durability.
If a part of the hierarchy is hard to understand, hard to trust, or hard to teach, its quality is too low.
