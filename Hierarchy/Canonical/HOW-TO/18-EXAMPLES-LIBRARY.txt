Examples library
================

This file is a curated library of example item shapes and path patterns.
Use it when you want to copy a proven form instead of inventing one.

Example 1: Basic same-layer work item
-------------------------------------
Use this for ordinary work that stays inside one taxonomy layer for a while.

Shape:
Layer X
  -> Workflow/01-Intake/20260415-basic.item.txt
  -> Workflow/02-Register/20260415-basic.item.txt
  -> Workflow/03-Identify/20260415-basic.item.txt
  -> Workflow/16-Build/20260415-basic.item.txt
  -> Workflow/18-Review/20260415-basic.item.txt
  -> Workflow/20-Approve/20260415-basic.item.txt

Good for:
- drafting
- implementation work
- local review before handoff

Example 5: Audit-trail heavy path
---------------------------------
Use this when history matters at every crossing.

Shape:
upstream layer late-state copy
upstream layer 24-Conveyor-Belt copy
upstream layer 25-Transfer copy
downstream layer 01-Intake live copy
trace companion file listing all of the above

Good for:
- regulated work
- teaching examples
- high-trust audit contexts

Example 6: Minimal live-item path
---------------------------------
Use this when simplicity matters more than preserved history.

Shape:
one active file that moves step by step with no extra copies

Good for:
- low-risk internal work
- short-lived tasks
- situations where clarity is improved by a single source of truth

Example 7: Terminal reference package
-------------------------------------
Use this when the goal is long-term retrieval, not active work.

Shape:
reference-oriented layer
  -> Workflow/01-Intake/20260415-reference.item.txt
  -> Workflow/26-Index/20260415-reference.item.txt
  -> Workflow/27-Archive-and-Reference/20260415-reference.item.txt
optional:
  -> trace file
  -> companion notes

Good for:
- stable knowledge
- preserved examples
- archived operational memory

Example 8: Human plus agent collaboration item
----------------------------------------------
Use this when a human shapes the meaning and an agent maintains motion and documentation.

Shape:
human creates item from `TEMPLATE-ITEM.txt`
agent uses `move-item.sh` for state accuracy
agent or human updates trace and HOW-TO docs if a recurring pattern appears

Good for:
- mixed manual and automated workflows
- repeatable operating systems
- collaborative maintenance

Example 9: Demo trace reference
-------------------------------
Use the existing live demonstration as a model.

Reference files:
- `/Applications/Hierarchy/Canonical/EXAMPLE-TRACE.txt`
- `20260415-conveyor-demo.item.txt` copies across repository, pull request, issue, and comment layers

Good for:
- onboarding
- teaching the conveyor model
- showing audit-trail style movement

Rule across all examples
------------------------
An example is good when it is easy to copy, easy to inspect, and faithful to the distinction between taxonomy and workflow.
