Glossary
========

This file defines the canonical vocabulary for the Canonical hierarchy.
Use these meanings consistently across items, guides, and scripts.

Archive-and-Reference
---------------------
The final workflow state for durable knowledge, memory, or long-term context.
This is step 27.

Audit trail
-----------
A preserved sequence of item copies across steps or layers that shows where the item has been.
Usually created with `move-item.sh --copy`.

Canonical
---------
The normalized version of the hierarchy that should be used for new work.
It is the structured successor to the Legacy tree.

Conveyor-Belt
-------------
The queue state before a cross-layer handoff.
This is step 24.
It is not the whole workflow, only the staging lane before transfer.

Current Path
------------
The workflow-relative location stored inside an item file.
Example: `Workflow/18-Review/`

Current Step
------------
The exact workflow state stored inside an item file.
Example: `18-Review`

Destination Layer
-----------------
The next taxonomy layer an item is expected to enter when it leaves its current layer.

Handoff
-------
The act of passing an item from one taxonomy layer to another.
The standard bridge is:
`23-Ready-to-Move -> 24-Conveyor-Belt -> 25-Transfer -> next layer 01-Intake`

Hierarchy
---------
The full system of taxonomy layers plus workflow states.
Not just folders, but a model of structure and motion.

HOW-TO
------
The documentation hub under `Canonical/HOW-TO/`.
This is the operating manual for the system.

Intake
------
The first workflow step where a new item enters a layer.
This is step 01.

Item
----
A file representing one unit of work, context, or reference moving through the hierarchy.
Recommended extension: `.item.txt`

Layer
-----
A taxonomy level that defines what kind of thing an item is associated with.
Examples: `05-Repository`, `13-Pull-Request`, `14-Issue`

Legacy
------
The original preserved concept chain.
It is kept for reference but is not the recommended working structure.

move-item.sh
------------
The helper script that moves or copies an item between workflow steps and updates its metadata.

Ready-to-Move
-------------
The approval state immediately before the conveyor lane.
This is step 23.

Source
------
The origin recorded inside an item file.
For cross-layer movement, this should usually point to the upstream step or file path.

Taxonomy
--------
The classification structure that answers: what kind of thing is this?
Taxonomy defines meaning, not state.

Template item
-------------
A file created from `TEMPLATE-ITEM.txt` that becomes a live artifact in the workflow.

Terminal reference
------------------
An item that has stopped being active work and is now durable context or knowledge.
Usually represented by step 27.

Trace file
----------
A companion file, such as `EXAMPLE-TRACE.txt`, that lists the path history of one item across states or layers.

Transfer
--------
The actual cross-layer movement point.
This is step 25.

Workflow
--------
The state machine inside each taxonomy layer.
Workflow answers: what state is the item in right now?
