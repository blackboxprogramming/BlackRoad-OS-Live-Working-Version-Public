Canonical hierarchy
===================

This tree is a layered operating model.
Each numbered directory is a taxonomy layer.
Each layer contains a Workflow/ directory that runs the same 27-step atomic sequence.

Layer guide
-----------
01-Enterprise         business or top-level operating context
02-Organization       company, team, or tenant container
03-User               person or principal acting in the system
04-Application        product or software surface
05-Repository         codebase or source container
06-Operations         action-oriented layer for repository activity
07-Fetch              retrieve remote state
08-Get                read specific resources
09-Push               send local changes upstream
10-Check              validate state, checks, or status
11-Workflows          automation and CI/CD definitions
12-Dependencies       packages, versions, and relationships
13-Pull-Request       review and merge unit
14-Issue              work item or defect record
15-Comment            discussion attached to work
16-Ongoing-Reference  durable notes, examples, or living context

Workflow rule
-------------
Taxonomy answers: what kind of thing is this?
Workflow answers: what exact state is it in right now?

Because every layer reuses the same 27 states, the whole tree scales without losing precision.
